# Lessons

- When the persona workbench is framed as "best output" or "authentic human texting," do not stop at a small export UI. The expected bar is a broad research-backed recommender with source coverage, code-switching/paralinguistic guidance, low-data augmentation strategy, and explicit authenticity diagnostics.
- When the user corrects a docs cleanup target, prioritize the newly named public reference and remove stale metadata before broadening the cleanup.
- When the user asks to analyze WhatsApp conversations without exports, treat direct read-only local sources as a first-class workflow. Keep export tooling as an optional compatibility path, not the required architecture.
- When naming experimental identity-modeling work, avoid grand claims such as "Living Brain." Use "Digital Self" as the honest umbrella, reserve "Mini-Me" for the playful single-person style workflow, and clearly distinguish product language from legacy package or schema identifiers.
- When the user asks for a README based on the project's current state, replace the inherited document instead of layering new framing over stale model catalogs, training recipes, or historical feature lists. Keep the README short and move implementation detail into linked docs.
